# Week 2 Reflection

## What I completed
- Cleaned the Titanic dataset using pandas.
- Built a data quality report for missing values, data types, duplicates, and unique values.
- Created a missing value strategy for `age`, `cabin`, and `embarked`.
- Engineered useful features such as `family_size`, `is_alone`, `title_group`, `age_group`, and `fare_group`.
- Used seaborn and matplotlib to visualize survival patterns.
- Created a focused Titanic EDA report.
- Published the Week 2 work on GitHub and shared a LinkedIn update.

## What I learned
- Data cleaning decisions should have reasons, not just code.
- Missing values can be handled differently depending on the column and amount of missing data.
- Feature engineering can make raw columns more useful for analysis.
- Survival rate can be calculated using the mean of a 0/1 target column.
- A report should focus on the strongest patterns, not every possible chart.

## What was hard
- Understanding when to drop a column versus filling missing values.
- Understanding why group-based age filling can be better than one overall median.
- Turning exploration into a clean final report.
- Keeping GitHub folders organized while working across multiple notebooks.
